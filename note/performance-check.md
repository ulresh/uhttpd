# Проверка производительности HTTP-сервера

У меня есть ряд претензий к стандартным библиотекам по работе со строками и буферами по производительности при работе с сетью, в том числе и в HTTP-сервере.

Кроме претензий также есть и понимание того, как сделать так, чтобы оно работало бысстрее.

Но понимание это одно, а хотелось бы точной оценки в цифрах.

Вот для этого и пишется приложение, которое будет одновременно содержать HTTP-сервер и клиент для проверки производительности при нагрузочном тестировании.

Давать нагрузку случайными запросами вариант плохой с той точки зрения, что не получится повторить те же условия для программы с другой реализацией строк.

Поэтому план тестирования будет генерироваться случайно, но записываться в файл, а после из этого файла воспроизводиться. То есть возникает второе приложение для создания планов тестирования.

Вариативность тестов буду создавать с использование словаря. Очень не хочется брать английский словарь, поэтому надо проверить как популярные браузеры реагируют на русские буквы в URI (и речь не про домен, там как раз понятно и проверка не требуется).

Кроме плана тестирования программе будет задано количество запросов (из плана), запускаемых одновременно.

Виды страниц для теста:
* список - вывести весь с разбивкой по страницым, добавить, вывести результат поиска с разбивкой по страницым
* статика
* компоновка 1-8 абзацев по параметрам в GET

Генератору планов на вход подаётся словарь и нобор абзацев текста.

Геренатор планов кроме планов также формирует дерево страниц.
При этом для одного дерева страниц может быть сформировано несколько планов тестирования.
